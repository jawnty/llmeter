tokmon

Dashboard API

Help

tokmon is a local token-usage dashboard for Claude Code and Codex. It reads the JSONL logs those tools already write, stores token counts in SQLite, and shows a live dashboard on localhost.

Install

npx tokmon install

Then open the dashboard. The installer copies tokmon into ~/.tokmon/app, creates a Python virtualenv, installs pinned dependencies, starts a launchd service on macOS, and opens the dashboard.

What Works Today

  • Claude Code logs from ~/.claude/projects/**/*.jsonl
  • Codex logs from ~/.codex/sessions/**/*.jsonl
  • Today totals, hourly bars, sessions, per-turn details, and live refreshes

LiteLLM And Security

tokmon is the dashboard/storage layer above local LLM tooling. This v1 does not proxy Claude Code or Codex traffic through LiteLLM; it reads local logs. For tools that need a proxy later, the expected expansion path is a LiteLLM-backed source with exact pinned versions. LiteLLM has had security-sensitive issues, so watch upstream advisories, do not expose proxies publicly, and run it at your own risk.

Stop Or Restart

npx tokmon stop
npx tokmon start
npx tokmon status

Settings

The defaults should work for most people. Advanced users can set TOKMON_PORT, TOKMON_HOST, TOKMON_DB_PATH, TOKMON_CLAUDE_GLOB, or TOKMON_CODEX_GLOB.
