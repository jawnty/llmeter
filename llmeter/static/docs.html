llmeter

Dashboard API

Help

llmeter is a local token-usage dashboard for Claude Code and Codex. It reads the JSONL logs those tools already write, stores token counts in SQLite, and shows a live dashboard on localhost.

Install

npx llmeter install

Then open the dashboard. The installer copies llmeter into ~/.llmeter/app, creates a Python virtualenv, installs pinned dependencies, starts a launchd service on macOS, and opens the dashboard.

What Works Today

  • Claude Code logs from ~/.claude/projects/**/*.jsonl
  • Codex logs from ~/.codex/sessions/**/*.jsonl
  • Today totals, hourly bars, sessions, per-turn details, and live refreshes

LiteLLM And Security

llmeter is the dashboard/storage layer above local LLM tooling. This v1 does not proxy Claude Code or Codex traffic through LiteLLM; it reads local logs. For tools that need a proxy later, the expected expansion path is a LiteLLM-backed source with exact pinned versions. LiteLLM has had security-sensitive issues, so watch upstream advisories, do not expose proxies publicly, and run it at your own risk.

Stop Or Restart

npx llmeter stop
npx llmeter start
npx llmeter status

Settings

The defaults should work for most people. Advanced users can set LLMETER_PORT, LLMETER_HOST, LLMETER_DB_PATH, LLMETER_CLAUDE_GLOB, or LLMETER_CODEX_GLOB.
